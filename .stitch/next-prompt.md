A mobile-optimized scanning interface for finders to quickly report items on campus using QR codes.

**DESIGN SYSTEM (REQUIRED):**
- Vibe & Mood: Minimal, aesthetic, modern, clean, software-as-a-service feel. Friendly but secure. No clutter. Generous whitespace, rounded components.
- Platform: Mobile (portrait-first)
- Theme: Light mode focused, with a soft cool-toned color palette.
- Primary Action: Soft Indigo / Deep Blue (#4F46E5) for primary buttons and accents.
- Background: Dark overlay for camera view, or light Pearl Gray (#F9FAFB) for UI elements.
- Surface / Cards: Pure white (#FFFFFF) with 16px rounded corners.
- Text Primary: Dark Slate (#1F2937).
- Text Secondary: Medium Gray (#6B7280).

**Page Structure:**
1. **Camera Viewfinder:** A large, full-width camera viewfinder section with a subtle "scanning" animation (e.g., a moving horizontal line). It should have rounded corners and a slight border.
2. **Help Text:** A small, centered text overlay above or below the viewfinder: "Point your camera at a FoundIt AI QR code located around campus."
3. **Quick Report Button:** A large, prominent "Report Found Item" pill button (#4F46E5) at the bottom.
4. **Manual Entry Link:** A secondary "Enter code manually" text link below the button.
5. **Recent Scans (Optional):** A small scrollable row or list showing the user's 2-3 most recent scans/reports for quick reference.

Make sure it feels native and fast. Since this is for students on the move, prioritize one-handed usability and high contrast.
